> To ensure modularity and interoperability, our first step must be defining the contract between Kiara and the underlying FOSS EDA tools (Yosys, nextpnr, openFPGALoader).
> 
> ### Architectural Decision Record (ADR-001): Toolchain Wrapper Interface
> 
> **Objective**: Define a unified, programmatic interface for invoking heterogeneous EDA tools.
> 
> **Proposed Design Principles**:
> 
> 1.  **Abstraction Layer**: Kiara must not invoke shell commands directly. We define an abstract `ToolchainExecutor` trait that each wrapper must implement.
> 2.  **Standardized Execution Context**: Every tool invocation must accept a standardized configuration object (defining input files, output directories, flags, and constraint files) and return a standardized result structure (exit code, stdout/stderr logs, artifact paths).
> 3.  **Asynchronous Handling**: The interface must support non-blocking execution, allowing the parent agent to monitor progress or cancel long-running synthesis/place-and-route tasks.
> 4.  **Serialization Contract**: All configuration and result data must be serializable (e.g., JSON) to facilitate inspection and logging by the AI-driven controller.
> 
> **Structural Proposal**:
> 
> *   **`Trait: ToolchainCommand`**: Defines the `execute()` method.
> *   **`Struct: CommandConfig`**: Encapsulates the execution parameters (path, arguments, environment variables).
> *   **`Struct: ExecutionResult`**: Encapsulates the outcome (success/failure, timing data, resource usage metrics).
> 
> **Why this approach?**
> By decoupling the *invocation logic* from the *tool-specific configuration*, we ensure that adding a new tool (e.g., a formal verification engine or a new vendor-specific place-and-route tool) only requires implementing the `ToolchainCommand` trait, without modifying the core Kiara engine.
> 
> **Does this architectural direction align with your vision for the Kiara suite?**
> 
> If validated, I will proceed to define the requirements for **ADR-002: The Project Orchestrator**, which manages the dependency graph between these toolchain tasks.
