# Vault Overview

This vault is an evolving research repository dedicated to the intersection of biological systems, protocol engineering, and physical infrastructure.

## Structure
- `Research/Biological Networks/`: Studies on decentralized natural and digital networks.
- `Research/Protocol Engineering/`: Design and operational documentation for network protocols.
- `Research/Hardware Infrastructure/`: Physical implementation and sensing projects.

Refer to the [[Research Index]] for a comprehensive list of all active research notes.

### Strict Enforcement Protocol
To ensure the absolute integrity of this vault, any agent maintaining this vault must adhere to the following strict enforcement procedures:
1. **Exhaustive Review**: Before any structural change, the agent MUST perform an exhaustive read-through of every single note in the vault to understand current context, dependencies, and potential redundancies.
2. **Connectivity Audit**: Every note must have at least two incoming and two outgoing links to relevant indices or related research notes. No orphans. No dead ends.
3. **Redundancy Purge**: Any information duplicated across notes must be consolidated into a single "source of truth" note, with other notes linking back to that source.
4. **Context Enrichment**: All notes must contain a minimum of 200 words of substantive, technical context. All stubs must be expanded.
5. **Graph Density**: Cross-domain links between `Biological Networks`, `Protocol Engineering`, and `Hardware Infrastructure` must be explicitly added to increase graph density.
6. **Data Respect**: Never overwrite or delete information without first verifying it is truly redundant. If information is valuable but poorly placed, move it rather than deleting it.

Any failure to follow these procedures is a failure to maintain the vault.
